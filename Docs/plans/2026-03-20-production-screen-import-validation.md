# Production screen import — validation notes (2026-03-20)

이 문서는 Task 7(임포트 리포트·고아 처리·문서/README)에 대한 **수동 검증 시나리오**를 정리합니다. 자동화된 Unity Play Mode 실행 없이도 재현 가능한지 위주로 작성했습니다.

## 사전 조건

- Unity **6000.0+** 프로젝트에 패키지가 들어가 있고, TMP Essentials·설정 에셋·Figma 문서 URL·Personal Access Token이 준비되어 있을 것.
- `UnityToFigma -> Sync Document`로 동기화할 수 있을 것.

## 시나리오

### 1) 인증 실패 (토큰/권한)

**준비**: 잘못된 Personal Access Token을 저장하거나, 만료·취소된 토큰을 사용합니다.

**기대**:

- 문서 다운로드 단계에서 실패합니다.
- 예외/로그에 **HTTP 상태 코드**와 응답 본문 일부가 포함되어 원인(401/403 등)을 추적하기 쉽습니다.
- 기존과 같이 에디터 대화 상자로 사용자에게 알립니다.

**비고**: 구현은 `FigmaApiUtils.GetFigmaDocument` 등 API 요청 실패 시 `FormatWebRequestFailure` 기반 메시지를 사용합니다.

### 2) 이미지 다운로드 실패

**준비**: 네트워크 차단, 잘못된 이미지 URL, 또는 Figma가 빈 URL을 돌려주는 서버 렌더 항목 등으로 `DownloadFiles` 경로가 실패하도록 합니다.

**기대**:

- 해당 항목은 **경고 로그**에 상세(요청 결과·경로)가 남습니다.
- `FigmaImportReport`의 **failed** 카운터가 증가하고, **Messages**에 항목별 한 줄 요약이 쌓입니다.
- 임포트가 끝나면 Console 요약 줄에 `failed=`이 0보다 클 수 있습니다.

### 3) 노드 이름/경로만 바뀜 (동일 노드 id 유지)

**준비**: Figma에서 화면 프레임의 **이름·계층 표시만** 바꾸고, 노드 id가 유지되는 변경을 합니다.

**기대**:

- 해당 노드는 문서 조회에 여전히 존재하므로, “문서에서 사라짐” 기준의 **고아 매니페스트 처리 대상이 아닙니다**.
- `PathUpdatePolicy`에 따라 **에셋 경로**는 유지되거나 최신 해석 경로로 바뀔 수 있습니다(기능 범위는 Task 1–6과 동일).

### 4) 노드 삭제·문서에서 제거 (id 소실)

**준비**: 이전에 임포트되어 매니페스트에 남아 있던 화면/컴포넌트 노드를 Figma에서 **삭제**하거나, 페이지 선택으로 이번 실행에서 해당 페이지를 제외하지 않은 상태에서 문서에서 사라지게 합니다.

**기대**:

- 이번 실행에서 해당 노드 id에 대해 prefab 저장(매니페스트 upsert)이 일어나지 않습니다.
- `MissingNodePolicy`가 **MarkAsOrphaned**(기본)이면 매니페스트 항목 상태가 **Orphaned**로 바뀌고, 요약의 **orphaned**가 증가합니다.
- **DeleteOnImport**이면 매니페스트 **행만** 제거되고 **manifestRemoved**가 증가합니다. **프로젝트 폴더의 .prefab 파일은 자동 삭제되지 않습니다.**

### 5) 페이지만 부분 임포트

**준비**: `OnlyImportSelectedPages`로 **일부 페이지만** 선택합니다.

**기대**:

- 선택되지 않은 페이지에만 있던 매니페스트 행은, 이번 실행의 “선택된 페이지” 범위 밖이므로 **고아 조정 대상에서 제외**됩니다(잘못 표시되지 않음).

## 자동 테스트

에디터 어셈블리에서 다음을 실행합니다.

- `FigmaImportReportTests` — 요약 문자열·리셋( manifestRemoved 포함 ).
- `FigmaImportManifestReconcilerTests` — 고아 표시·삭제 정책·페이지 범위·노드 존재 시 스킵.

## 알려진 한계 (이번 범위)

- **Skipped** 카운터는 아직 생성기 전 구간에서 광범위하게 채워지지 않을 수 있습니다(요약에는 0이 나올 수 있음).
- 성공한 임포트마다 대화 상자를 띄우지 않고, 실패·고아·매니페스트 제거가 있을 때만 추가 알림을 띄웁니다.
